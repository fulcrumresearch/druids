## Running on MorphCloud

We run on a Morph instance. Key info is in `~/.env`:
```
MORPH_INSTANCE_ID=morphvm_xxx
```

## Getting the MCP URL

The server runs on port 8000. Use the SDK to get the exposed MCP URL:

```python
import os
from orpheus.core.morph import get_client

client = get_client()
inst = client.instances.get(os.environ["MORPH_INSTANCE_ID"])
for svc in inst.networking.http_services:
    if svc.port == 8000:
        print(f"{svc.url}/mcp/")
```

If port 8000 is not exposed yet:
```python
url = await inst.aexpose_http_service("orpheus-server", 8000)
print(f"{url}/mcp/")
```

If port conflict, kill existing process:
```bash
ss -tlnp | grep 8000  # Find PID
kill <pid>
```

## GitHub PR Flow

Agents create branches named `orpheus/{execution_slug}` and open PRs. The agent calls the `submit` MCP tool after creating a PR, which persists PR metadata and marks the execution complete.

When a reviewer comments on the PR, GitHub sends a webhook to `POST /webhooks/github`. The server looks up the execution by repo and PR number, resumes it (transitioning from "submitted" back to "running"), and re-prompts the root agent with the feedback text. The agent addresses the feedback, pushes new commits, and calls `submit` again. This cycle repeats for each round of review.

The webhook handles two GitHub event types:
- `issue_comment.created`: PR comments
- `pull_request_review.submitted`: Reviews with state `changes_requested` or `commented` (approvals are ignored)

Bot comments and reviews are ignored to prevent feedback loops. The webhook requires `GITHUB_WEBHOOK_SECRET` to be configured for HMAC-SHA256 signature verification.

## Slug Generation

Tasks get music-themed slugs: `{adjective}-{piece}` (e.g., "gentle-nocturne", "electric-sonata"). Generated by `utils/slugs.py` using the `coolname` library.
Executions add program name and attempt number: `{task_slug}-{program}` or `{task_slug}-{program}-{attempt}` (e.g., "gentle-nocturne-claude", "gentle-nocturne-claude-2").

## CLI Development

### Typer Patterns (`cli/orpheus/main.py`)

Commands use Typer with sub-app grouping:

```python
app = typer.Typer(name="orpheus", help="...", no_args_is_help=True)
app.add_typer(auth, name="auth")

@app.command()
def exec(
    spec_file: Path = typer.Argument(..., help="Path to spec file"),
    program: list[str] | None = typer.Option(None, "--program", "-p", help="..."),
    all_programs: bool = typer.Option(False, "--all", help="Run all available programs in parallel"),
):
```

Entry point: `orpheus = "orpheus.main:app"` in `cli/pyproject.toml`.

### Error Handling

```python
try:
    data = client.get_task(task_slug)
except NotFoundError as e:
    print_error(str(e))
    raise typer.Exit(1)
except APIError as e:
    print_error(f"Error: {e}")
    raise typer.Exit(1)
```

Authentication checks use a helper: `get_authenticated_client()` returns an `OrpheusClient` or exits with an error message.

### HTTP Client (`cli/orpheus/client.py`)

`OrpheusClient` wraps `httpx.Client` with `base_url` from config and Bearer token in headers. Methods like `create_task()`, `get_task()`, `stop_task()` handle HTTP status codes and raise `APIError` or `NotFoundError`.

### Output Formatting (`cli/orpheus/display.py`)

Rich library for terminal output. Three helpers: `print_success()` (green check), `print_error()` (red X), `print_warning()` (yellow !). Use `console.print()` with Rich markup for formatted output.

### GitHub Device Flow (`cli/orpheus/auth.py`)

OAuth device flow implementation with `match`/`case` for error dispatching. `poll_for_user_access_token()` loops with `time.sleep()` until the user authorizes or the code expires.
